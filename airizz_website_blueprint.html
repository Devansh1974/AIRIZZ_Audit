Section 02
Animation & Motion Spec
Every animation listed here. Use GSAP for complex scroll + timeline effects. CSS transitions for micro-interactions. Respect prefers-reduced-motion by wrapping all GSAP in a media query check.
⚡
Animation Philosophy
Animations serve two purposes: guide attention and signal quality. Every animation must have a reason. No animation longer than 600ms on UI elements. Scroll reveals: 400ms ease-out. Hero entrance: 1.2s orchestrated timeline. Always: ease-out for entrances, ease-in for exits. GPU-accelerated only (transform, opacity — never animate width/height/top/left directly).
Hero Section
Canvas Particle Network
Animated SVG/Canvas network of nodes and connecting lines in the hero background — mimics the AIRIZZ logo's network topology. Nodes pulse softly. Lines draw between nearby nodes. On mouse-move: nearby nodes repel slightly (magnetic effect). Low opacity so it doesn't compete with text.
vanilla canvas API
Hero Section
Staggered Text Reveal
H1 headline: each word fades up with 80ms stagger delay. Subheadline: fades in 300ms after headline. CTA button: scales from 0.95 → 1.0 with a subtle glow pulse on entrance. Total entrance sequence: ~1.2 seconds orchestrated with GSAP timeline.
GSAP timeline + SplitText
Scroll
Fade-Up on Scroll (Global)
Every section heading, card, and feature block fades up (translateY: 30px → 0, opacity: 0 → 1) as it enters the viewport. Cards in grids stagger with 80ms delay per item. Implemented via IntersectionObserver for performance — no scroll event listeners.
IntersectionObserver API
Scroll
Parallax Section Backgrounds
Radial glow/mesh background elements in hero and stats sections move at 0.4× scroll speed relative to content (parallax). Creates depth without complexity. Only applied to background decorative elements — never to text or interactive elements.
GSAP ScrollTrigger
Stats Section
Number Counter Animation
Metric numbers count up from 0 to final value over 1.8 seconds when they enter the viewport. Easing: power2.out. Numbers display with comma formatting. Triggered once per page load — not repeated on re-scroll. Add "+" suffix where applicable.
GSAP numTo / CountUp.js
Hover States
Card Hover Lift + Glow
Service cards, case study cards: on hover → translateY(-4px), border-color shifts to var(--border2), a subtle inset box-shadow glow (0 0 20px rgba(0,194,203,0.12)) appears. Transition: 200ms ease-out. On mouse-leave: reverses in 150ms. Gives the "floating card" premium feel.
CSS transition
Hover States
CTA Button Shimmer
Primary CTA buttons: a shimmer/shine sweep animation plays on hover (pseudo-element with linear-gradient at 45°, translateX from -100% → 200% over 500ms). The gradient matches the brand colors. Click: subtle scale(0.97) press-down effect. Returns to normal on mouse-up.
CSS keyframes + :hover
Navigation
Nav Link Underline Slide
Desktop nav links: a teal underline (2px, gradient) slides in from left on hover, slides out on mouse-leave. Active page: underline is always visible. Nav bar: transitions background opacity from 0 to 0.92 after 80px scroll. Mobile menu: stagger-reveals links one by one (60ms delay per link).
CSS transform + transition
Logo Strip
Infinite Client Logo Marquee
Client logos / industry names scroll horizontally in an infinite loop (CSS animation, duplicated content for seamless wrap). Speed: ~40s per full cycle. On hover: pauses the marquee. Logos are grayscale at rest, full color on hover. Smooth with CSS transform — no JS needed.
CSS @keyframes (translateX)
Industry Tabs
Tab Content Crossfade
On tab switch: outgoing content fades out (opacity 1→0, 150ms), incoming content fades in + slides up slightly (opacity 0→1, translateY 8px→0, 250ms). Active tab indicator: teal underline slides horizontally to new active tab position (GSAP or CSS transition on left/width). Smooth, not jumpy.
CSS transition + JS class toggle
Testimonials
Auto-Rotating Carousel
Testimonial cards auto-advance every 5 seconds with a smooth horizontal slide transition (transform: translateX). Drag/swipe enabled on mobile (touch events). Dots indicator at bottom. On hover: pauses auto-rotation. Navigation: prev/next arrow buttons with the teal accent style.
CSS transitions + touch events
Process Steps
Step Line Draw Animation
The vertical connecting line between process steps "draws" downward as each step enters the viewport (SVG stroke-dashoffset animation). Each step circle pulses once when it becomes active. Steps reveal sequentially with 200ms delay. Creates a sense of progression and momentum.
SVG stroke-dashoffset + GSAP
♿
Accessibility Rule
Always wrap GSAP animations in: if (!window.matchMedia('(prefers-reduced-motion: reduce)').matches). Users who have reduced motion enabled in OS settings should see instant reveals, not animated ones. This is WCAG 2.1 AA compliance.
